CLI-Integrated Sync Implementation Plan¶

For Claude: REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

Goal: Replace the separate sync service/timer path with a single cross-platform CLI-managed sync loop that keeps heartbeat execution isolated from risky update work.

Architecture: marrow run stays the only long-running service entrypoint and gains a lightweight periodic sync supervisor. The supervisor launches marrow sync-once as a subprocess, consumes a small result contract, reloads safe changes in-process, and exits for unsafe changes so the existing service manager can restart cleanly.

Tech Stack: Python 3.12, Typer CLI, asyncio, subprocess execution, JSON state files, launchd/systemd service templates, pytest

Step 3: Write the minimal implementation

Move the useful sync.sh behavior into Python helpers, keeping execution conservative:

  • fetch and compare remote revision
  • reject dirty worktrees
